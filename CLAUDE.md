## Project Overview

**Bookmarker.dev** is a free, lightweight bookmark manager application built with Nuxt 4, Vue 3, TypeScript, and Tailwind CSS. It allows users to create, edit, reorder, and customize bookmarks with a simple interface. All data persists locally in the browser via localStorage.

The project is currently on Nuxt 4 (compatibility date 2025-07-15) with a TypeScript setup and modern tooling.

## Architecture Overview

### State Management (Pinia Store)

**File**: `app/stores/useAppStore.ts`

The app uses a single Pinia store that manages:
- **Bookmarks**: Array of bookmark objects with `name`, `url`, `favicon`, and `key`
- **Settings**: User customization (background color, text color, background image + properties)
- **UI State**: Whether the settings modal is open

Actions in the store handle updates and delegate localStorage persistence via `useHelpers()`. Settings changes automatically persist to localStorage.

### Components Structure

**Location**: `app/components/`

Key components:
- **the-bookmark.vue**: Individual bookmark card. Shows favicon, name, and action buttons (edit/delete)
- **edit-bookmark.vue**: Modal form for creating/editing bookmarks. Validates URLs and fetches favicons
- **the-modal.vue**: Reusable modal wrapper for dialogs
- **global-settings.vue**: Settings form for background customization, text color, etc.
- **the-input.vue**, **the-select.vue**: Reusable form controls

### Composables (Shared Logic)

**File**: `app/composables/useHelpers.ts`

Exports all utilities for:
- **localStorage persistence**: `loadStorageValue()`, `setStorageSetting()`, `setStorageBookmarks()`, `deleteStorageBookmark()`, `reOrderStorageBookmarks()`
- **URL validation**: `isUrlValid()` (uses `URL.canParse()`)
- **Color utilities**: `getOppositeColor()` (hex → inverted hex)
- **Storage checks**: `checkLocalStorage()`

All localStorage keys are: `'bookmarks'` and `'settings'`

### Root Component

**File**: `app/app.vue`

Main layout combining:
- Draggable grid of bookmarks (via `vuedraggable`)
- Add bookmark button
- Settings toggle button (bottom-left)
- App logo display (bottom-right)
- Two modals: edit/add bookmark modal and settings modal

Uses CSS variables bound to Pinia state for dynamic theming (background-color, text-color, background image + CSS properties).

### Styling

**Location**: `app/assets/css/`

- **_main.css**: Entry point imported in `nuxt.config.ts`
- **body.css**: Global styles and resets
- Tailwind CSS v4 integrated via `@tailwindcss/vite` plugin
- Dynamic CSS bindings in components use `v-bind()` syntax for reactive theming

## Data Persistence

The app uses **localStorage** exclusively. No backend or database—everything is client-side:

```javascript
localStorage.setItem('bookmarks', JSON.stringify([...])) // Array of bookmark objects
localStorage.setItem('settings', JSON.stringify({...}))  // Settings object
```

On mount, `app.vue` loads both and populates the store. All mutations sync back to localStorage via store actions.

## Common Patterns

**Adding a new bookmark**:
1. User clicks add button → sets `activeBookmark` to empty object with `new: true`
2. Modal opens with `edit-bookmark.vue`
3. Form validates URL via `isUrlValid()` and saves via `setStorageBookmarks()`
4. Store action updates `appStore.bookmarks` (which triggers re-render)

**Customizing theme**:
1. User changes color in `global-settings.vue`
2. Store action (`handleUpdateBackgroundColor`, etc.) updates store and calls `setStorageSetting()`
3. CSS v-bindings reactively update background styles on `.application` div

**Reordering bookmarks**:
1. User drags bookmark card
2. `draggable` component updates `appStore.bookmarks` array
3. `@change` event calls `reOrderStorageBookmarks()` to persist new order
